# Debian Packaging Patch Mirror

This directory tracks the Debian-specific patch queue for OpenCC so upstream
maintainers can review Debian packaging deltas before a release.

Current baseline:

- Debian source package: `opencc 1.3.1+ds1-3`
- Patch index:
  `https://udd.debian.org/patches.cgi?src=opencc&version=1.3.1%2Bds1-3`

Intent:

- Keep Debian packaging deltas visible to upstream maintainers.
- Identify patches that can be replaced by upstream-supported build options.
- Make release-time Debian coordination easier.

This directory is not authoritative Debian packaging. Debian maintainers should
continue to use the Debian Salsa repository and the Debian source package as the
source of truth.

Source package handling:

- `CMakeLists.txt` excludes `/packaging/` from CPack source archives.
- `package.json` uses an npm `files` whitelist, so this directory is not packed
  into the npm package.
- GitHub's autogenerated "Source code" archives include all tracked files and
  cannot be filtered by repository metadata. Use the CPack-generated source
  archive for release assets when this directory must be excluded.

CPack source package profiles:

- The default `OPENCC_SOURCE_PACKAGE_PROFILE=full` keeps the existing source
  package behavior.
- `OPENCC_SOURCE_PACKAGE_PROFILE=opencc` produces a C++-focused source archive
  named `opencc-<version>-opencc-src.*`. It excludes Node.js, Python, Doxygen
  documentation sources, Jieba plugin sources, benchmark-only dependencies, and
  packaging metadata. Core C++ tests and bundled GoogleTest are kept so
  maintainers can run `ctest` after configuring with `ENABLE_GTEST=ON`.
- `OPENCC_SOURCE_PACKAGE_PROFILE=opencc-jieba` produces a Jieba plugin-focused
  source archive named `opencc-jieba-<version>-src.*`. It keeps the OpenCC C++
  core and the Jieba plugin sources, but excludes Node.js, Python, Doxygen
  documentation sources, benchmark-only dependencies, and packaging metadata.
  Core C++ tests, bundled GoogleTest, and Jieba integration tests are kept so
  maintainers can run `ctest` after configuring with `ENABLE_GTEST=ON`.

Example:

```sh
cmake -S . -B build-opencc-src -DOPENCC_SOURCE_PACKAGE_PROFILE=opencc
cmake --build build-opencc-src --target package_source

cmake -S . -B build-opencc-jieba-src -DOPENCC_SOURCE_PACKAGE_PROFILE=opencc-jieba
cmake --build build-opencc-jieba-src --target package_source
```

CPack source archives are generated from the filesystem, not from `git archive`.
Create release archives from a clean checkout so untracked local files cannot be
included accidentally.
