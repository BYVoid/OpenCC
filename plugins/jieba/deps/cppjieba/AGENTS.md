# Repository Instructions

- After every code update, assess whether a new cppjieba release should be published.
- After committing changes, push the commit to the remote repository.
- After publishing a new cppjieba release, create issues in nodejieba, gojieba, and simhash recommending that they update to the latest cppjieba version.
- When there are no other active tasks in this repository, check for open issues and pull requests that need attention, and help address them when possible.
